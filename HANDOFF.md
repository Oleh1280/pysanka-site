## 🎯 Перші 5 завдань для тебе, Claude Code

### Завдання 1: Перевір синхронізацію локальної копії з репо

```bash
cd ~/Pysanka  # (або де у Олега локальний клон)
git status
git branch -a
git log --oneline -10
git fetch origin
```

Подивись чи локальна копія відповідає GitHub. Покажи Олегу результат.

### Завдання 2: Залий поточний стан у репо

У моєму ZIP `pysanka-site-handoff.zip` є **всі найсвіжіші файли** включно з:
- README.md
- CLAUDE.md (постійний контекст про проєкт)
- HANDOFF.md (цей файл)
- ROADMAP.md (детальний план Фаз 1-4)
- noindex meta-теги в HTML

Олег розпакує цей ZIP. Ти допоможеш йому:

1. **Замінити** старі файли в локальній копії на ці нові
2. **Створити гілку `develop`** локально
3. **Закомітити** все в `develop`
4. **Запушити** `develop` на GitHub
5. **Створити PR** `develop → main` через `gh pr create`
6. **Не міржити** PR одразу — нехай Олег подивиться на staging URL

### Завдання 3: Налаштуй Branch Protection на main

Через `gh` API:

```bash
gh api repos/Oleh1280/pysanka-site/branches/main/protection \
  -X PUT \
  -F required_pull_request_reviews.required_approving_review_count=0 \
  -F required_pull_request_reviews.dismiss_stale_reviews=false \
  -F enforce_admins=false \
  -F required_status_checks=null \
  -F restrictions=null
```

Або підкажи Олегу зробити це через веб-інтерфейс, якщо API не спрацює.

Перевір що захист працює: спробуй `git push origin main` — має зафейлитися.

### Завдання 4: Налаштуй Branch Deploy на Netlify

```bash
netlify status   # перевір що залогінений в правильний site
netlify open     # відкриє site в браузері
```

Через UI або CLI: увімкни Branch Deploys для `develop`. Очікуваний результат:
- Production URL: `https://[name].netlify.app` (з main)
- Staging URL: `https://develop--[name].netlify.app` (з develop)

Якщо Олег ще не перейменував site — запропонуй йому 3-5 варіантів красивих імен:
- pysanky-syrotiuk
- pysanky-koliada
- pysanka-master-galyna
- pysanky-handmade-ua
- pysanka-syrotiuk-piatnychuk

### Завдання 5: Перевір Phase 0 працює

Поясни Олегу як перевірити:
1. Зайти на staging URL — побачити сайт з усіма правками
2. Спробувати зробити пробний коміт у develop — переконатися що Netlify деплоїть автоматично

Коли Phase 0 повністю запрацює — переходь до **ROADMAP.md** для Фази 1.

## 🚨 Важливі застереження

### Що НЕ роби без явної згоди Олега

1. **Не виконуй force push** в main, навіть якщо здається необхідним
2. **Не видаляй файли в `images/`** — це фото справжніх робіт майстрині, цінний контент
3. **Не комітуй секрети** — API ключі завжди йдуть в Netlify Environment Variables, ніколи в код
4. **Не змінюй стилістику** — палітра, шрифт, структура продумані з Олегом
5. **Не переходь на React/Vue/Next.js** — Олег обрав чистий HTML/CSS/JS свідомо
6. **Не публікуй сайт публічно** для третіх осіб без узгодження. Майстриня має побачити перша.

### Що Олег часто плутає (нормально для не-розробника)

- **GitHub vs Git** — давай м'які пояснення коли потрібно
- **Гілки vs форки** — пояснюй простою аналогією (гілка = варіант одного дерева, форк = окреме дерево)
- **Pull vs Push** — pull = «тягни до себе», push = «штовхай назовні»
- **Локальна копія vs GitHub** — це окремі копії, синхронізуються вручну

### Що Олег **очень добре** розуміє

- UX і дизайн — він UX/Product людина
- Архітектурні рішення на high level
- Бізнес-вимоги (ми будуємо це для майстрині)
- Емоційну сторону проєкту (це особиста історія, не просто магазин)
